VCDとは（短く）

VCD は、価値と責任が、文脈（章）の変化を越えて引き渡されるための前提条件を定義します。 これは方法論ではなく、境界を置く理由そのものです。

BOA が「どう分けるか」を扱うのに対し、 VCD は「なぜ分けなければならないのか」を扱います。

設計は、どの「時間」を扱ってきたのか

私たちはこれまで、無意識のうちに Phase（開発フェーズ） や Version（バージョン） という区切りで、システムを設計してきました。 それらは進捗や変更を管理するには十分でしたが、 人の経験が積み重なり、意味が変わり、責任が移ろっていく時間を扱うには十分ではありませんでした。

VCD は、この扱われてこなかった時間を 章（Chapter） という設計単位として引き受けます。

Phase / Version / Chapter（VCD）の違い

VCD は Phase や Version を否定しません。
それらでは扱いきれなかった 「時間の中で変わる意味と責任」を、設計の対象にします。

観点 Phase（開発フェーズ） Version（バージョン） Chapter（VCD）
主な目的 作業を区切る 変更を識別する 価値を継続させる
扱う時間 予定された時間 リリース時点 経験が積み重なる時間
人の経験 含まれない 含まれない 含まれる
意味の変化 想定しない 差分として扱う 前提として受け止める
責任の所在 プロセス外 記録外 章ごとに明示される
終わり方 完了 更新 次の章への引き渡し
設計の関心 進捗管理 変更管理 責任の移ろい

※ 太字は VCD が新しく持ち込んだ視点です。Phase / Version は有効ですが、VCD は「その外側の時間（章）」を扱います。

VCDesignの中でのVCDの位置

VCD は VCDesign の「背骨」です。
IDG / RP / BOA は、この哲学レイヤの上で「境界」を具体化する入口です。

  • IDG — 曖昧な入力を境界で止める 「何を通してはいけないか」の基準は VCD が与える。
  • RP — Meaning → Responsibility の昇格を守る 「どの判断が価値を壊すか」の基準は VCD が決める。
  • BOA — Fact / Meaning / Responsibility を分離する どこに境界を置くべきかの優先順位を VCD が示す。
