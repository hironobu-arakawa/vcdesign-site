VCDesign エコシステム

境界が決める設計へ。

VCDesign はフレームワークではありません。 「変化しても壊れないために、どこに境界を置くか」を扱うエコシステムです。 4つの入口（IDG / RP / BOA / VCD）から、必要な境界だけを選んで使えます。

IDG — インターフェース決定可能性ゲート

曖昧な入力はここで止まります。 説明できないもの、責任が曖昧なものは境界を通過できません。

IDG を開く

RP — 解釈昇格プロトコル

解釈（Meaning）が責任（Responsibility）に昇格する瞬間を守るゲート。 「いつ誰が決めたか」を曖昧にしません。

RP を開く

BOA — Boundary-Oriented Architecture

Fact / Meaning / Responsibility を境界で分離する構築方法。 現場でもすぐ使える実践的な入口です。

BOA を開く

VCD — Value Continuity Design

VCDesign の哲学的コア（基盤）。 なぜ境界が必要なのか、 なぜ価値と責任は文脈を越えて残らなければならないのかを扱います。

VCD を開く

Library（衛星リポジトリ）

現場ノート、データ形状、責務の原則など、 VCDesign / BOA の周辺にある実践的な資料群です。 ここは「読む場所」ではなく「参照先を決める場所」です。

Library を開く
