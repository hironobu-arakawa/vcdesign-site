Step 1: Load Prerequisite Files

In this demo, you first have the chat load "prerequisite design files".
Subsequent interactions proceed sharing these premises.

A. Questioning Value Continuity (VCD)

Ask not for the "correct answer" but "where it is likely to weather/decay".
A question to understand the core of VCDesign (design to continue value).

https://raw.githubusercontent.com/hironobu-arakawa/vcdesign-core/main/vcdesign_core.yaml regarding the system I am designing, please point out "future risks" from the perspective of VCDesign (Value Continuity Design). * Once loaded, simply answer "Understood".

B. Want to know the content of this site's library

Load the following URL first and treat it as a premise for the conversation. https://vcdesign.org/library/library.yaml * Once loaded, simply answer "Understood the structure".

C. Want to avoid "ambiguity" in API or IF design (IDG)

Use when you do not want to leave "who decides" or "how far is guaranteed" ambiguous in API specifications or interface design.

Load the following repository and grasp the implementation patterns and definitions of IDG (Interface Determinability Gate). https://github.com/hironobu-arakawa/interface-determinability-gate Once loaded, summarize the main points in 3 lines.

Step 3: What Returns?

AI that identifies VCDesign will return "boundary pointers" rather than simple code fixes.

  • NG Example "It is more convenient to make this parameter optional."
  • VCDesign Answer "Making this parameter optional makes 'who decides' ambiguous, violating IDG Ownership. Make it mandatory or separate the caller."
