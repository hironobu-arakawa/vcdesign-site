5 Premises

Philosophical premises upon which the entire VCDesign depends.

  1. 1. Meaning depends on context Value wavers when context changes.
  2. 2. Judgment requires human responsibility AI can propose candidates, but cannot hold responsibility.
  3. 3. Boundaries limit meaning Meaning changes when crossing a boundary. It cannot be carried over completely.
  4. 4. Indeterminate things must not be fixed Treating changeable things as "facts" causes breakage.
  5. 5. Preservation is a result of flow Preservation is not the goal, but a byproduct of flow.

These are not "ideals to be protected",
but conditions that will cause breakage if ignored.

Where are these premises embodied?

VCD only sets the premises.
BOA / RP / IDG are the entry points to embody them as boundaries.

  • IDG — Stop vague inputs at the boundary VCD provides the criteria for "what must not pass".
  • RP — Protect the promotion of Meaning → Responsibility VCD decides the criteria for "which judgment breaks value".
  • BOA — Separate Fact / Meaning / Responsibility VCD indicates the priority of where boundaries should be placed.
