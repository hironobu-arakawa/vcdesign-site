What is VCD (Briefly)

VCD defines the prerequisites for value and responsibility to be handed over across changes in context (Chapters). This is not a methodology, but the reason itself for placing boundaries.

While BOA deals with "how to separate", VCD deals with "why we must separate".

What "Time" has design been dealing with?

Until now, we have unconsciously designed systems using Phase or Version as delimiters. These were sufficient for managing progress and changes, but not sufficient for dealing with time where human experience accumulates, meaning changes, and responsibility shifts.

VCD accepts this previously unaddressed time as a design unit called Chapter.

Difference between Phase / Version / Chapter (VCD)

VCD does not deny Phase or Version.
It targets "meaning and responsibility that change within time", which those could not fully handle.

Position of VCD within VCDesign

VCD is the "backbone" of VCDesign.
IDG / RP / BOA are entry points that embody "boundaries" on top of this philosophy layer.

  • IDG — Stop vague inputs at the boundary VCD provides the criteria for "what must not pass".
  • RP — Protect the promotion of Meaning → Responsibility VCD decides the criteria for "which judgment breaks value".
  • BOA — Separate Fact / Meaning / Responsibility VCD indicates the priority of where boundaries should be placed.
